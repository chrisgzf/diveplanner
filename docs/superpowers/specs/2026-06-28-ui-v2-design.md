# DivePlanner UI v2 — Design Spec
**Date:** 2026-06-28
**Status:** Approved

---

## Overview

This spec covers a second wave of UI changes driven by user feedback after the v1 implementation. The changes fall into five groups: layout & navigation, dark mode, calendar cell display & hover, trip drawer improvements, and leave/holiday calculation fixes.

---

## 1. Layout & Navigation

### 1.1 Sticky header row

`Nav` is already `sticky top-0 z-20`. `LeaveBalanceBar` is made sticky at `top-14` (below the nav, which is ~56px tall) with `z-10`. Both bars remain pinned while scrolling.

### 1.2 Responsive top nav (no bottom tab bar)

The fixed mobile bottom tab bar is removed. All navigation items live in the top nav bar at all screen sizes.

- **Desktop (≥ md):** `[Icon + Text]` for each item.
- **Mobile (< md):** `[Icon only]` — text labels hidden.

Nav items in order: Planner, Locations, Settings, Share, Theme toggle (sun/moon icon). `Settings` and `Share` move out of the `actions` prop and into the main nav item list. `Nav` no longer accepts an `actions` prop.

### 1.3 Compact 3-column calendar grid

`CalendarView` wraps its month list in a CSS grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`.

All months render at once; the page scrolls normally through them. No lazy loading or pagination.

`MonthGrid` gets a compact mode: smaller heading (`text-sm`), smaller day cells (`text-xs`, tighter `aspect-square`), tighter internal spacing.

### 1.4 Calendar window

`calendarWindow(now: Date): { year: number; month: number }[]` in `lib/dates.ts` replaces the old `monthsWindow`. It returns every month from the current month through December of `currentYear + 1`. Maximum 24 months, minimum 13.

The Settings dialog continues to show exactly two leave-budget rows: `currentYear` and `currentYear + 1`.

### 1.5 Static split panel on desktop (trip drawer)

On desktop (`≥ lg`), `PlannerPage` uses a static 60/40 flex split — calendar always ~60% width, right column always ~40% width, regardless of selection state. This avoids a layout shift/reflow when a trip panel opens. The right column shows:

- `TripsOverview` (all trips, sorted by start date, clickable to edit) when no date range or trip is selected.
- `TripPanel` inline when a range is selected or a trip is being edited.

On mobile (`< lg`): `TripPanel` renders inside a Shadcn `Sheet` overlay (existing behaviour) since there's no room for a static side pane.

The drawer form content is extracted into a standalone `TripPanel` component shared by both layouts.

When the drawer is open on desktop, the calendar container narrows, and the 3-column grid naturally collapses toward 2 columns.

## 2. Dark Mode

### 2.1 Approach

Dark mode is implemented via a `.dark` class on `<html>`, which overrides CSS custom properties. No Tailwind `dark:` prefixes are used — all colours are already semantic variables.

Default: **dark mode**.

`Settings` gains `theme: 'dark' | 'light'` (default `'dark'`). `App.tsx` applies:

```ts
useEffect(() => {
  document.documentElement.classList.toggle('dark', settings.theme === 'dark')
}, [settings.theme])
```

### 2.2 Colour tokens

A `--surface-elevated` token is added to replace all hardcoded `bg-white` on inputs, selects, textareas, dialogs, and the drawer panel.

| Token | Light | Dark |
|---|---|---|
| `--surface` | `#F2F7F8` | `#0D1E27` |
| `--surface-elevated` | `#FFFFFF` | `#122B38` |
| `--ink` | `#0B1F2A` | `#E8F4F6` |
| `--muted` | `#5B7682` | `#7BA3AE` |
| `--line` | `#D7E3E6` | `#1E3545` |
| `--primary` | `#0E7C86` | `#14A8B5` |
| `--fair` | `#D9A406` | `#F5C030` |

`--good`, `--poor`, `--closed`, and all trip-type colours are unchanged — they are readable on both backgrounds.

### 2.3 Shadcn variable bridge

Added once to `index.css` inside both `:root` and `.dark`, mapping Shadcn's expected variable names to our tokens:

```css
--background: var(--surface);
--foreground: var(--ink);
--card: var(--surface-elevated);
--card-foreground: var(--ink);
--popover: var(--surface-elevated);
--popover-foreground: var(--ink);
--border: var(--line);
--input: var(--line);
--ring: var(--primary);
--muted-foreground: var(--muted);
```

**Implementation note (Task 6):** In Tailwind v3, CSS custom properties defined in `index.css` do not automatically produce utility classes. All bridge variables (`background`, `foreground`, `card`, `card-foreground`, `popover`, `popover-foreground`, `primary-foreground`, `border`, `input`, `ring`, `muted-foreground`) are explicitly registered in the `colors` map in `tailwind.config.ts` so that utilities like `bg-card`, `text-foreground`, `border-border`, `ring-ring`, and `text-muted-foreground` exist and are picked up by the vendored `ui/*` components. `surface-elevated` is also registered at this step.

### 2.4 Audit: remove hardcoded `bg-white`

All raw inputs, selects, and textareas in `TripDrawer`, `SettingsDialog`, `BookingChecklist`, and `AddLocationDialog` replace `bg-white` with `bg-surface-elevated`. Shadcn Dialog and Sheet components pick up `bg-card` automatically via the bridge.

**Implementation note (Task 6):** `DialogContent` in `src/components/ui/dialog.tsx` and the `sheetVariants` base class in `src/components/ui/sheet.tsx` were updated from `bg-background` to `bg-card`. This ensures those overlay surfaces render at the elevated level (`--surface-elevated`) rather than at the page background level (`--surface`), giving them the correct visual lift in both light and dark modes.

## 3. Calendar Cell Display & Hover

### 3.1 Trip day cell — full colour highlight

`CalendarView` builds `covered: Map<ISODate, Trip>` (date → owning trip) via a new `coveredTripMap(trips: Trip[]): Map<ISODate, Trip>` helper in `lib/overlap.ts`, replacing the existing `coveredDays` Set. The old `coveredDays` helper is removed. `covered` flows down to `MonthGrid` → `DayCell` as `coveredByTrip?: Trip`.

`DayCell` applies the trip's colour at high opacity when covered:
- `fun-dive` → `bg-fun-dive/80 text-white`
- `liveaboard` → `bg-liveaboard/80 text-white`
- `course` → `bg-course/80 text-white`
- `non-dive` → `bg-non-dive/60 text-ink`

Start day: `rounded-l-md rounded-r-none`. End day: `rounded-r-md rounded-l-none`. Mid-range days: `rounded-none`. Single-day trips: full rounding.

### 3.2 Holiday indicator

Replace the bottom dot with an amber underline: `border-b-2 border-fair` on the day number. Visible against both light and dark backgrounds using the `--fair` token.

### 3.3 Holiday name data

`parseHolidays` is updated to return `HolidayEntry[]`:

```ts
interface HolidayEntry {
  date: ISODate
  name: string
}
```

The store's `holidays` slice changes from `Record<string, ISODate[]>` to `Record<string, HolidayEntry[]>`. A companion `holidayNameMap(cache: Record<string, HolidayEntry[]>): Map<ISODate, string>` helper returns date → name for display. `holidaySetFromCache(cache: Record<string, HolidayEntry[]>): Set<ISODate>` is updated to read `.date` from each entry — its signature and return type are unchanged, only its parameter type widens.

### 3.4 Hover tooltip

Every `DayCell` renders a Radix Tooltip (Shadcn `Tooltip` component) on hover. Content is assembled from a `DayMeta` record built once in `CalendarView`:

```ts
interface DayMeta {
  holidayName?: string
  trip?: Trip
  goodLocs: string[]   // location names rated 'good' for that calendar month
  fairLocs: string[]   // location names rated 'fair' for that calendar month
}
```

`dayMetaMap: Map<ISODate, DayMeta>` is computed in `CalendarView` from the merged location list (from store), the holiday name map, and the covered-trip map. It is passed into `MonthGrid` → `DayCell`.

Tooltip content:

```
Good for diving          ← omitted if empty
· Bunaken · Tioman

Fair                     ← omitted if empty
· Koh Tao

Holiday: National Day    ← omitted if no holiday
Planned: Tioman LW 2026  ← omitted if no trip; label truncated to 30 chars
```

No tooltip renders if `DayMeta` is entirely empty (plain uncovered weekday with no holiday).

## 4. Trip Drawer

### 4.1 Form field improvements

| Field | Change |
|---|---|
| Trip name placeholder | `"e.g. Malapascua May 2026"` |
| Start date label | `"Start (inclusive)"` |
| End date label | `"End (inclusive)"` |
| Status default (create mode) | `'wishlist'` (was `'planned'`) |
| Accommodation booking label placeholder | `"e.g. Homely Hostel"` |
| Dive shop booking label placeholder | `"e.g. Blahblah Divers"` (unchanged — this one is correct) |

### 4.2 Location "Other" option

`Trip` gains an optional field:

```ts
interface Trip {
  // ... existing fields ...
  customLocation?: string  // used when no known locationId is selected
}
```

`LocationPicker` adds an "Other..." entry at the bottom of the combobox list. When selected, an inline text input appears for the custom location name (stored in `customLocation`). `locationId` is set to `undefined` when "Other..." is active.

Display: `TripBlock` and the drawer header show `customLocation` when `locationId` is absent.

### 4.3 Seasonality panel in drawer

A "Dive conditions" section appears below the date inputs when `type !== 'non-dive'`. It lists locations grouped by their rating for the trip's months (union of all calendar months in the date range):

```
Good for diving
· Bunaken · Tioman · Nusa Penida

Fair
· Koh Tao · Malapascua
```

Only good and fair locations are shown. Poor/closed locations are omitted. Computed from `mergedLocations` (base + overrides) in the store.

### 4.4 Segmented leave breakdown

`segmentDays(start, end, holidayNames: Map<ISODate, string>): Segment[]` in `lib/leave.ts` groups consecutive days by type:

```ts
type SegmentKind = 'leave' | 'holiday' | 'weekend'

interface Segment {
  kind: SegmentKind
  startDate: ISODate
  endDate: ISODate
  label?: string  // holiday name when kind === 'holiday'
  days: number    // count of calendar days in this segment
  leaveDays: number  // leave days consumed (0 for weekend/holiday segments)
}
```

The existing single-line `"Leave: 2026: 3d"` display is replaced by a compact segmented list:

```
Thu 30 Apr – Fri 1 May   Holiday (Labour Day)
Mon 4 May                1 day leave
Sat–Sun 2–3 May          Weekend

Total: 1 day leave · 5 days trip
```

The dive estimate override remains at the bottom of this block.

### 4.5 Per-day leave exclusion

Some weekdays inside a trip range aren't actually taken as leave (e.g. flying out after work on a Friday). `Trip.excludedLeaveDates?: ISODate[]` stores which specific leave-kind dates are excluded from the tally. This is threaded through `lib/leave.ts`'s `leaveDays`/`leaveDaysInRange`/`leaveDaysByYear`/`leaveUsedByYear` as an optional `excluded: Set<ISODate>` parameter, so both the trip drawer's own total and the global leave balance bar reflect it.

UI (`components/LeaveBreakdown.tsx`): each `leave`-kind segment (which may already span multiple consecutive weekdays, per §4.4's grouping) renders as **one row** with a single checkbox — checked means every day in that block counts as leave. Unchecking it excludes the whole block. Multi-day blocks also get a `▸`/`▾` expand toggle to reveal one checkbox per individual date, for the rarer case where only some days within a block should be excluded. Weekend/holiday segments are unaffected (no checkbox — they never count as leave regardless).

## 5. Leave & Holiday Calculation

### 5.1 Substitute holiday logic

`applySubstituteHolidays(entries: HolidayEntry[]): HolidayEntry[]` in `lib/holidays.ts` chains substitute holidays for weekend public holidays:

- Process weekend holidays in chronological order.
- For each holiday on Saturday or Sunday, find the next calendar day (Monday, Tuesday, …) that is not already occupied by an original holiday or a previously assigned substitute.
- Add that date as a substitute entry with `name: "<original name> (substitute)"`.
- Original weekend entries are kept (they appear in tooltips on their actual date).

Example: Sat holiday → substitute Mon. Sun holiday same weekend → substitute Tue (Mon already taken).

This function is called inside `useHolidays` after each successful Nager.Date fetch, before storing entries in the session cache.

### 5.2 `calendarWindow` replaces `monthsWindow`

```ts
function calendarWindow(now: Date): { year: number; month: number }[]
```

Returns months from `{ year: now.getFullYear(), month: now.getMonth() + 1 }` through `{ year: now.getFullYear() + 1, month: 12 }`. The old `monthsWindow` is removed (or deprecated if tests reference it).

## 6. Unchanged

- Data model fields not listed above are unchanged.
- Share encoding (`lz-string`) is unchanged; `customLocation` is naturally included in the serialised `Trip`.
- Location explorer page layout is unchanged.
- All 57 existing tests must continue to pass; new behaviour gets new tests.
